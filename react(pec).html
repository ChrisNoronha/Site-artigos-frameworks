Vantagens do React JS:
‬‬‬‬‬‬‬‬


-Fácil de ler e se utilizar: Ele vem com um bom suprimento de documentação, tutoriais e recursos de treinamento. Qualquer desenvolvedor com experiência em JavaScript pode entender facilmente e começar a criar aplicativos da web usando o React em poucos dias. É a parte V (view) no modelo MVC (Model-View-Controller) e referido como "um dos frameworks JavaScript.
-A criação de aplicativos da Web dinâmicos se torna mais fácil: Criar uma aplicação web dinâmica especificamente com strings HTML era complicado porque requer uma codificação complexa, mas o React JS resolveu esse problema e tornou isso mais fácil. Ele fornece menos codificação e dá mais funcionalidade. Ele faz uso do JSX (JavaScript Extension), que é uma sintaxe específica que permite citações HTML e sintaxe de marca HTML para renderizar subcomponentes específicos. Ele também suporta a construção de códigos legíveis por máquina.
-Componentes reutilizáveis: O código reutilizável ajuda a tornar seus aplicativos mais fáceis de desenvolver e manter. Esses componentes podem ser aninhados com outros componentes para permitir que aplicativos complexos sejam construídos a partir de blocos de construção simples. ReactJS usa mecanismo baseado em DOM virtual para preencher dados em HTML DOM. O DOM virtual funciona rápido, pois altera apenas os elementos individuais do DOM, em vez de recarregar o DOM completo todas as vezes.
‬‬‬‬‬‬‬Melhoria de desempenho: Melhora o desempenho devido ao DOM virtual. O DOM é uma API de plataforma cruzada e programação que lida com HTML, XML ou XHTML. A maioria dos desenvolvedores enfrentou o problema quando o DOM foi atualizado, o que diminuiu o desempenho do aplicativo. O ReactJS resolveu esse problema introduzindo o DOM virtual. O React Virtual DOM existe inteiramente na memória e é uma representação do DOM do navegador da web.
O benefício de ter uma biblioteca JavaScript: Hoje, o ReactJS é escolhido pela maioria dos desenvolvedores da web. É porque está oferecendo uma biblioteca JavaScript muito rica. A biblioteca JavaScript fornece mais flexibilidade para os desenvolvedores da Web escolherem a maneira que desejam.

Desvantagens do React JS:


-Diversas mudanças:‬‬‬‬‬‬‬ Devido ao rápido ritmo de crescimento, o ambiente está sempre mudando, o que dificulta a adaptação dos programadores. Com uma atualização e aceleração tão rápidas, uma boa documentação é difícil de ser obtida. O ReactJS apenas lida com as camadas de IU do aplicativo, portanto você precisará usar outras ferramentas para outras partes do desenvolvimento. Se há muitos modelos e eles se sobrepõem, o JSX pode ser confuso.
-Documentação ruim: É outro contras que são comuns para tecnologias de atualização constante. Tecnologias React atualizando e acelerando tão rápido que não há tempo para fazer a documentação adequada. Para superar isso, os desenvolvedores escrevem instruções por conta própria com a evolução de novos lançamentos e ferramentas em seus projetos atuais.
-Ver Parte: ReactJS cobre apenas as camadas de interface do usuário do aplicativo e nada mais. Portanto, você ainda precisa escolher algumas outras tecnologias para obter um conjunto completo de ferramentas para desenvolvimento no projeto.
-JSX como uma barreira: ReactJS usa JSX. É uma extensão de sintaxe que permite misturar HTML com JavaScript. Essa abordagem tem seus próprios benefícios, mas alguns membros da comunidade de desenvolvimento consideram o JSX uma barreira, especialmente para novos desenvolvedores. Os desenvolvedores reclamam de sua complexidade na curva de aprendizado. ‬‬‬‬‬ ‬‬‬
